Rather than patch each screen individually, added a persistent global nav bar (`src/components/TopNav.tsx`, shown on every screen except the Hub, which already has its own header) with a "◆ DICE THRONE" home link and a contextual "← Back" button, plus a current-screen breadcrumb. Back is backed by a real navigation history stack added to `store.tsx` (`goBack`/`canGoBack`, tracked outside persisted state so it always starts empty on load) rather than a single previous-screen pointer, so e.g. Hub → Missions → Trackers → Back → Back correctly walks back through Missions before returning to the Hub instead of ping-ponging between the last two screens. `App.tsx` now wraps the routed screen in a `flex:1, overflow:auto` container beneath the nav bar so existing screens (which all assume `height:100%` from their parent) don't need any changes. Also added `.claude/launch.json` so the dev server can be previewed via the `run` skill going forward. Verified `tsc -b` passes clean and walked the full flow in the browser: PvP (previously a dead end) now shows Back/breadcrumb, Status Effects and Rules likewise, and the two-level Missions → Trackers → Missions → Hub back-chain resolves correctly at each step with no console errors.
- **2026-07-16** — Fixed the app always reopening on whatever screen was last visited instead of the Hub. There's no URL routing at all — `state.screen` was saved to `localStorage` in full and restored verbatim on load, so e.g. leaving the app on Missions Setup meant every future visit (including the root URL) skipped the Hub entirely and dropped straight back into Setup. The Hub already has a "Continue Session" resume bar built for exactly this purpose. Fixed in `store.tsx` by forcing `screen: 'hub'` on load while still restoring everything else (round, heroes, crisis clock, PvP/Adventures/Perks progress), so the Hub is reachable again and "Continue Session" still has accurate data. Verified in the browser: seeded `localStorage` with a persisted `setup` screen, reloaded, confirmed it lands on the Hub with the session data intact.
- **2026-07-16** — Tightened up loose typing in `store.tsx`: replaced `pvp: any`, `adv: any`, and `perkPlayers: any[]` with real `PvpState`/`PvpPlayer`, `AdvState`/`AdvSession`, and `PlayerPerks` interfaces derived from how `PvP.tsx`, `Adventures.tsx`, `AdventuresSetup.tsx`, and `Perks.tsx` actually use them. `Perks.tsx` and `Adventures.tsx` had each declared their own local copy of these shapes (`PlayerPerks`, `Session`) — deduped those to import from `store.tsx` instead. Along the way, fixed a real bug in `AdventuresSetup.tsx`'s campaign-start seeding: `Array(8).fill({...})` was filling all 8 scenario rows with references to the *same* object, so editing one row's Win/Loss/Score would silently mutate all the others; switched to `Array.from({ length: 8 }, () => ({...}))` for independent objects. Verified `tsc -b` passes clean and exercised PvP (start a match), Adventures (seed campaign, toggle Win/Loss on one row only), and Perks in the browser with no console errors.
